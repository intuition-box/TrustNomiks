## Recommended Next Steps

1. **Verify remote state:** Run `SELECT tablename, policies FROM pg_policies WHERE schemaname = 'public'` in the Supabase SQL Editor to see what policies actually exist.
2. **If tables lack RLS:** Add policies for at least `tokens` (the root table — ownership on all CRUD operations).
3. **Priority order for RLS enablement:** `tokens` > `profiles` > child tables (which are already protected by RPCs for writes, but remain open for direct reads).
4. **Export page reads:** The export pipeline does `SELECT *` on all tables. If RLS is not enabled on those tables, any authenticated user can export any token's data. This may be intentional (all data is public within the org) or a gap depending on the access model.
